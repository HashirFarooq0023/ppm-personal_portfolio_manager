# Personal Portfolio Manager (PPSM) - Frontend Architecture & Design Detail

This document provides a comprehensive, high-resolution breakdown of the PPSM Frontend. It explores the "Crystal Glass" design system, component architecture, state management patterns, and interactive UX strategies.

---

## 1. Vision & Design Philosophy

The PPSM Frontend is designed to feel like a **Premium Trading Terminal**. We utilize the "Crystal Glass" design language, characterized by:
- **Sophisticated Glassmorphism**: High-blur background filters, subtle borders, and layered translucency.
- **Dynamic Energy**: Animated backgrounds (data streams, neural pulses) that give the app a "living" feel.
- **Data-Centric Typography**: Use of the `IBM Plex Mono` font for numeric data to ensure tabular alignment and readability.
- **Monochromatic Sophistication**: A dark-mode first approach with vibrant primary accents (Honeydew/Emerald for gains, Rose/Crimson for losses).

---

## 2. Technology Stack

- **Framework**: React 18 (with Vite for lighting-fast builds).
- **Styling**: Tailwind CSS + Framer Motion (for high-fidelity animations).
- **UI Components**: Shadcn UI (Customized for glassmorphism).
- **Authentication**: Clerk (User management and session handling).
- **Data Fetching**: TanStack Query v5 (Caching, polling, and auto-refetching).
- **Charts**: Lightweight-charts (High-performance financial visualization).
- **Routing**: React Router v6.

## 4. Component Deep Dive

### charts/CandlestickChart.tsx
A high-performance wrapper around the TradingView Lightweight Charts library.
- **Customization**: Injected with the Crystal Glass theme (grid line colors, background transparency).
- **Responsiveness**: Uses a `ResizeObserver` to ensure the chart fits its container perfectly across all device sizes.
- **Data Handling**: Transforms raw time-series data into OHLC (Open, High, Low, Close) formats required by the library.

### Market/StockDetailView.tsx
The most complex interaction node in the application.
- **Unified Logic**: Handles state for both "Market Watch" view and "Owned Portfolio" view.
- **Interactive Ledger**: Allows users to see every transaction for a stock and delete individual entries.
- **Contextual Actions**: Dynamically shows "Buy", "Sell", or "Restore" buttons based on whether the stock is in the active portfolio or the bin.

### ui/detail-card.tsx
A fundamental atom of the design system.
- **Layout**: Features a small header-cap for the label and a large mono-tabular value.
- **Semantic Highlighting**: Automatically applies green/red coloring based on the `highlight` prop, used extensively for P&L tracking.

---

## 5. State Management: TanStack Query

We rely heavily on TanStack Query for a "server-state-first" approach.

### Polling Strategies
- **Market Overview**: Polled every 10 seconds to keep the dashboard tickers and top movers updated.
- **Portfolio Data**: Polled every 5 minutes (or manually refetched after a trade) to keep account totals accurate.
- **Stock History**: Cached by symbol to avoid redundant network requests when switching between different stock views.

### Local vs. Server State
- **Server State**: Market prices, holdings, user profile.
- **Local State**: Search queries, selected time ranges, UI toggle states (Sidebar open/closed).

---

## 6. Design System: index.css

The global stylesheet defines the "Living" aspects of the UI.

### CSS Variables
We define a curated set of HSL tokens:
- `--background`: Deep navy/black base.
- `--primary`: Vibrant electric green (Emerald-400).
- `--surface-glass`: Translucent white/black with 10% opacity.
- `--border-glass`: Subtly visible edges (20% opacity).

### Animations
- **Glass Pulse**: A subtle breathing effect applied to active cards.
- **Neural Stream**: A background animation for the landing page that mimics data flow across a network.
- **Smooth Transitions**: All hover states and layout changes use `cubic-bezier` transitions for a premium feel.

---

## 7. Routing & Security

The app uses **React Router v6** with nested layouts.
- **Public Routes**: `/`, `/sign-in`, `/sign-up`.
- **Protected Routes**: Everything under the `/dashboard` shell. We use Clerk's `<SignedIn>` and `<SignedOut>` components to handle redirects automatically.
- **Layout Persistence**: The `AppLayout.tsx` ensures the Sidebar and Navbar remain mounted during navigation, preventing UI flicker.

---

## 8. Data Formatting (src/data/mockData.ts)

Precision is key in financial apps.
- **formatPKR()**: Standardizes currency display across the app, ensuring proper comma separation and decimal precision.
- **formatNumber()**: Compresses large volumes (e.g., 1.2M) for cleaner table views.
- **generateCandleData()**: Used to provide immediate visual feedback even if the backend scraper has a momentary delay.

---

## 9. Performance Optimization

- **Component Memoization**: Heavy components like charts use `useMemo` and `useCallback` to prevent unnecessary re-renders during frequent price updates.
- **Asset Optimization**: All icons are SVGs (Lucide-React), and fonts are optimized for low latency.
- **Code Splitting**: Vite automatically chunks the application to ensure the initial bundle size remains small.
